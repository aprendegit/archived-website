]]> Por: admin https://aprendegit.com/ignorando-ficheros-en-git-parte-ii-prioridades/#comment-1182 Mon, 21 Oct 2013 19:00:36 +0000 https://aprendegit.com/?p=217#comment-1182 En respuesta a Angel Jimenez.

Hola Ángel:

Lo que debes hacer es decirle a git que deje monitorizar ese fichero. Para ello, utiliza el comando git-update-index:

git update-index --assume-unchanged app.php

Si pasado un tiempo quieres volver a monitorizar los cambios, basta con que hagas

git update-index --no-assume-unchanged app.php

Este método tiene una pega y es que cada persona que clona el repositorio deberá ejecutar estos comandos. Yo creo que una solución más interesante, que es la que yo suelo usar es la la siguiente:

  • Creas un fichero app.php de referencia, digamos app_ref.php
  • Creas un fichero app.php que no metes en el repositorio, ignorándolo desde el principio en .gitignore
  • Metes la información pre-establecida en app_ref.php y cuando cualquier persona se clona el repositorio, copia este fichero a app.php y lo personaliza si lo necesita. Podrías incluso crear un script para ello.

Quizás esta forma de proceder te resulte un poco más cómoda. Espero haberte ayudado.
